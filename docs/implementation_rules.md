# Implementation Rules

Implement only approved requirements and testcases in this repository.

Rules:

- Do not add unapproved features.
- Write tests first.
- Then write implementation.
- Run tests after each meaningful change.
- Measure progress only by passing testcases.
- If a testcase is unclear, stop and ask a question.
- At the end, create a review report:
  - implemented requirements
  - passed tests
  - failed tests
  - changed files
  - open points
